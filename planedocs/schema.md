# 单机飞机实时轨迹数据库方案

> 本文档面向当前项目。轨迹数据事实来源是已存在的 `qar_sample`，不是旧项目的 `system_position` 和 `flight_status`。首期不要求新建轨迹事实表。

## 1. 结论

当前项目已经具备实时轨迹所需的核心表：

- `data_record`：保存 UDP 报文公共元数据、飞机注册号、机型、航司、航班号、起降机场、接收时间。
- `qar_sample`：保存 QAR 采样点，包括经纬度、高度、速度、航向、姿态和剩余航程。

因此首期数据库方案是：

1. 复用 `qar_sample` 作为轨迹点事实表。
2. 复用 `data_record` 补齐飞机、航司和原始报文元数据。
3. 增加必要查询索引，支撑“最新点”和“当前航段轨迹”。
4. 机场和航司名称先用后端静态映射；只有需要后台维护时再新增字典表。

## 2. 当前表结构依据

当前迁移文件：

```text
backend/src/main/resources/db/migration/V4__create_simulator_business_tables.sql
```

已经创建：

```sql
CREATE TABLE qar_sample (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    record_id uuid NOT NULL REFERENCES data_record(id) ON DELETE CASCADE,
    sample_at timestamptz NOT NULL,
    source_time_text varchar(16) NOT NULL,
    flight_no varchar(20) NOT NULL,
    origin varchar(4) NOT NULL,
    destination varchar(4) NOT NULL,
    air_ground_status varchar(16) NOT NULL,
    altitude_ft numeric(10,2),
    computed_air_speed_kt numeric(10,3),
    ground_speed_kt numeric(10,3),
    latitude double precision,
    longitude double precision,
    track_angle_deg numeric(7,3),
    heading_deg numeric(7,3),
    pitch_deg numeric(7,3),
    roll_deg numeric(7,3),
    left_fuel_qty numeric(14,3),
    right_fuel_qty numeric(14,3),
    center_fuel_qty numeric(14,3),
    low_fuel_warning boolean,
    distance_to_go_nm numeric(12,3),
    destination_eta_text varchar(32),
    frame_count bigint NOT NULL,
    created_at timestamptz NOT NULL DEFAULT now()
);
```

现有索引：

```sql
CREATE INDEX idx_qar_flight_sample ON qar_sample (flight_no, sample_at);
CREATE INDEX idx_qar_frame_count ON qar_sample (frame_count);
CREATE INDEX brin_qar_sample_at ON qar_sample USING brin (sample_at);
```

这些索引能支撑基础查询，但实时页面更偏向“最新点”和“倒序时间窗口”，建议补充下面的索引。

## 3. 首期推荐新增索引

如数据库中已有等价索引，不需要重复创建。

```sql
CREATE INDEX idx_qar_sample_sample_at_desc
    ON qar_sample (sample_at DESC)
    WHERE latitude IS NOT NULL
      AND longitude IS NOT NULL;

CREATE INDEX idx_qar_sample_flight_sample_desc
    ON qar_sample (flight_no, sample_at DESC)
    WHERE latitude IS NOT NULL
      AND longitude IS NOT NULL;

CREATE INDEX idx_qar_sample_active_recent
    ON qar_sample (sample_at DESC, flight_no)
    WHERE latitude IS NOT NULL
      AND longitude IS NOT NULL
      AND ground_speed_kt > 100;
```

用途：

| 索引 | 用途 |
| --- | --- |
| `idx_qar_sample_sample_at_desc` | 快速找到最近有效 QAR 点 |
| `idx_qar_sample_flight_sample_desc` | 查询当前航班最近 24 小时轨迹 |
| `idx_qar_sample_active_recent` | 快速判断当前是否处于飞行中 |

注意：PostgreSQL 部分索引条件不能包含动态时间，例如 `now() - interval '5 minutes'`，新鲜度窗口应放在查询条件里。

## 4. 可选新增字典表

首期不强制建表。若希望机场、航司名称由数据库维护，可新增轻量字典表。

### 4.1 `airport_dictionary`

```sql
CREATE TABLE airport_dictionary (
    code varchar(4) PRIMARY KEY,
    name varchar(100) NOT NULL,
```

## 5. 不推荐新增的表

首期不要新增以下旧项目表：

| 表 | 原因 |
| --- | --- |
| `system_position` | 当前 QAR 已提供经纬度、高度、航向和速度，重复建表会造成双事实源 |
| `flight_status` | 当前 QAR 已携带航班、起降机场、剩余航程和 ETA 文本 |
| `flight_track` | 当前航段可由 `qar_sample` 按时间窗口动态查询 |
| `flight_segment` | 历史回放不是首期目标 |
| Redis 缓存表或状态表 | 单机低频轮询暂不需要 |

如果后续接入真实飞机数据且数据量显著增长，再评估独立时序表或 TimescaleDB。当前演示阶段不提前复杂化。

## 6. 查询设计

### 6.1 查询最新有效点

```sql
SELECT
    q.*,
    r.aircraft_registration_no,
    r.aircraft_model,
    r.airline_code
FROM qar_sample q
JOIN data_record r ON r.id = q.record_id
WHERE q.sample_at >= now() - interval '5 minutes'
  AND q.latitude IS NOT NULL
  AND q.longitude IS NOT NULL
ORDER BY q.sample_at DESC
LIMIT 1;
```

后端拿到最新点后再判断：

- `ground_speed_kt > 100`：飞行中，继续查轨迹。
- `ground_speed_kt <= 100` 或为空：当前无活跃轨迹。

### 6.2 查询当前航班轨迹

```sql
SELECT
    q.*
FROM qar_sample q
WHERE q.flight_no = #{flightNo}
  AND q.sample_at >= #{windowStart}
  AND q.sample_at <= #{latestSampleAt}
  AND q.latitude IS NOT NULL
  AND q.longitude IS NOT NULL
ORDER BY q.sample_at ASC;
```

其中：

- `windowStart = latestSampleAt - 24 hours`。
- 查询结果由后端等距抽样到约 `720` 点，并保留首点和最新点。
- 如果需要切分活跃航段，可在 Java 层从结果末尾向前查找最近一次 `ground_speed_kt <= 100`，只保留之后的点。

### 6.3 查询当前航班元数据

优先直接使用最新点关联到的 `data_record`：

```sql
SELECT
    aircraft_registration_no,
    aircraft_model,
    airline_code,
    flight_no,
    origin,
    destination
FROM data_record
WHERE id = #{recordId};
```

如果 `data_record` 中部分字段为空，以 `qar_sample.flight_no`、`origin`、`destination` 兜底。

## 7. 数据质量规则

| 规则 | 说明 |
| --- | --- |
| 经纬度为空 | 不参与地图轨迹 |
| 经纬度越界 | 已由现有 check 约束阻止 |
| `sample_at` 乱序 | 查询必须 `ORDER BY sample_at ASC` |
| `frame_count` 重复 | 当前表没有唯一约束，首期允许；必要时以后加去重逻辑 |
| `flight_no` 变化 | 以最新有效点的航班号作为当前航班 |
| 机场代码 | 当前模拟器使用四字 ICAO，API 不应改成三字码 |
| 高度单位 | `altitude_ft` 按 ft 展示，不再写成 meters |

## 9. 验收 SQL

模拟器发送后，可用以下查询确认数据链路：

```sql
SELECT count(*) FROM qar_sample;
```

```sql
SELECT
    flight_no,
    sample_at,
    latitude,
    longitude,
    altitude_ft,
    ground_speed_kt,
    track_angle_deg
FROM qar_sample
WHERE latitude IS NOT NULL
  AND longitude IS NOT NULL
ORDER BY sample_at DESC
LIMIT 5;
```

```sql
SELECT
    q.flight_no,
    min(q.sample_at) AS start_at,
    max(q.sample_at) AS end_at,
    count(*) AS point_count
FROM qar_sample q
WHERE q.flight_no = (
    SELECT flight_no
    FROM qar_sample
    WHERE latitude IS NOT NULL
      AND longitude IS NOT NULL
    ORDER BY sample_at DESC
    LIMIT 1
)
GROUP BY q.flight_no;
```

验收通过标准：

- `qar_sample` 有持续新增记录。
- 最近记录有经纬度、地速、航向。
- 当前航班至少有两条可画轨迹的点。
- 查询不依赖旧项目表。
