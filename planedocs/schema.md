# 飞机实时轨迹展示数据库 Schema

> 本文档基于 `docs/SREC.md` 设计，仅覆盖飞机实时轨迹展示的最小可复用数据库结构。目标数据库为 PostgreSQL，字符编码使用 UTF-8。本文只写建表依据和字段规格，不提供实现代码。

## 1. 建表范围

| 表名 | 必需 | 作用 |
|---|---|---|
| `system_position` | 必需 | 系统位置数据表，存储飞机轨迹点，是实时位置和轨迹线的事实表 |
| `flight_status` | 必需 | 飞行状态数据表，补充航班号、机场、剩余时间和剩余距离 |

不单独建立 `flight_track` 或 `flight_segment` 表。实时航段由后端按 `system_position.air_id`、`time_stamp` 和 `ground_speed` 动态计算。

## 2. 全局约定

| 项 | 约定 |
|---|---|
| 数据库 | PostgreSQL 13+ |
| 时间戳 | `time_stamp` 使用 Unix 秒，类型为 `BIGINT` |
| 接收时间 | `received_at` 使用 `TIMESTAMPTZ`，默认地面系统当前时间 |
| 主键 | `id` 使用 `BIGINT GENERATED BY DEFAULT AS IDENTITY`，允许后端显式写入 Snowflake ID |
| 经纬度 | `latitude` 范围 `[-90, 90]`，`longitude` 范围 `[-180, 180]` |
| 机场代码 | `or_airport` / `des_airport` 长度保留 10；建议统一存三字码，若源数据为四字码则由入库或映射层转换 |
| 航班号 | `flight_num` 不允许 `NULL`，未知时使用空字符串 |
| 活跃判断 | `ground_speed > 100` Knots 且时间戳在后端新鲜度窗口内 |

## 3. 表：`system_position`

### 3.1 表说明

`system_position` 存储系统位置数据，每一行是一架飞机在某一时刻的位置和姿态。实时快照取每架飞机最新一行；完整轨迹取指定飞机一段时间内的多行，并按 `time_stamp` 升序排列。

### 3.2 字段

| 字段名 | PostgreSQL 类型 | 必填 | 默认值 | 约束 | 说明 |
|---|---|---:|---|---|---|
| `id` | `BIGINT` | 是 | identity | 主键 | 主键 ID，允许后端显式写入 |
| `data_id` | `INTEGER` | 是 | 无 | `CHECK data_id = 217` | 数据 ID，固定为 `0xD9` |
| `time_stamp` | `BIGINT` | 是 | 无 | `CHECK time_stamp > 0` | 数据发送时间戳，Unix 秒 |
| `air_id` | `VARCHAR(50)` | 是 | 无 | 非空字符串 | 飞机注册号/机尾号 |
| `flight_num` | `VARCHAR(50)` | 是 | `''` | 不为 `NULL` | 航班号，未知时为空字符串 |
| `received_at` | `TIMESTAMPTZ` | 是 | 当前时间 | 无 | 地面系统接收时间 |
| `utc` | `VARCHAR(30)` | 否 | `NULL` | 无 | 原始 UTC 时间字符串 |
| `latitude` | `DOUBLE PRECISION` | 否 | `NULL` | 空值或 `-90 <= latitude <= 90` | 纬度，单位 degrees |
| `longitude` | `DOUBLE PRECISION` | 否 | `NULL` | 空值或 `-180 <= longitude <= 180` | 经度，单位 degrees |
| `altitude` | `DOUBLE PRECISION` | 否 | `NULL` | 无 | 高度，单位 meters |
| `true_heading` | `DOUBLE PRECISION` | 否 | `NULL` | 空值或 `0 <= true_heading < 360` | 真航向，单位 degrees |
| `pitch_angle` | `DOUBLE PRECISION` | 否 | `NULL` | 空值或 `-180 <= pitch_angle <= 180` | 俯仰角，单位 degrees |
| `roll_angle` | `DOUBLE PRECISION` | 否 | `NULL` | 空值或 `-180 <= roll_angle <= 180` | 横滚角，单位 degrees |
| `body_pitch_rate` | `DOUBLE PRECISION` | 否 | `NULL` | 无 | 俯仰率，单位 degrees/s |
| `body_roll_rate` | `DOUBLE PRECISION` | 否 | `NULL` | 无 | 横滚率 |
| `heading_angular_rate` | `DOUBLE PRECISION` | 否 | `NULL` | 无 | 航向角速率，单位 degrees/s |
| `ground_speed` | `DOUBLE PRECISION` | 否 | `NULL` | 空值或 `ground_speed >= 0` | 地速，单位 Knots |

### 3.3 约束

| 约束名 | 类型 | 字段 | 规则 |
|---|---|---|---|
| `pk_system_position` | 主键 | `id` | 全表唯一 |
| `ck_system_position_data_id` | 检查 | `data_id` | 必须等于 `217` |
| `ck_system_position_time_stamp` | 检查 | `time_stamp` | 必须大于 `0` |
| `ck_system_position_air_id_not_blank` | 检查 | `air_id` | 去除空白后不能为空 |
| `ck_system_position_latitude` | 检查 | `latitude` | 为空或在 `[-90, 90]` |
| `ck_system_position_longitude` | 检查 | `longitude` | 为空或在 `[-180, 180]` |
| `ck_system_position_ground_speed` | 检查 | `ground_speed` | 为空或大于等于 `0` |
| `uq_system_position_packet` | 唯一 | `data_id, air_id, flight_num, time_stamp` | 防止同一飞机同一航班同一秒重复写入位置点 |

### 3.4 索引

| 索引名 | 字段 | 类型 | 用途 |
|---|---|---|---|
| `idx_system_position_airid_ts_desc` | `air_id, time_stamp DESC` | B-tree | 单机最新航段、历史轨迹查询 |
| `idx_system_position_ts_airid_desc` | `time_stamp DESC, air_id` | B-tree | 活跃快照数据库兜底，先限制时间窗口再按飞机聚合 |
| `idx_system_position_flightnum_ts_desc` | `flight_num, time_stamp DESC` | B-tree | 按航班号排查或扩展查询 |
| `idx_system_position_active_speed` | `time_stamp DESC, air_id`，过滤条件 `ground_speed > 100` | 部分索引 | 加速活跃飞机快照查询 |

## 4. 表：`flight_status`

### 4.1 表说明

`flight_status` 存储飞行状态数据。实时展示中，它不是轨迹点来源，而是补充 `system_position` 中缺少的航班业务信息，例如起飞机场、目的机场、剩余时间和剩余距离。

### 4.2 字段

| 字段名 | PostgreSQL 类型 | 必填 | 默认值 | 约束 | 说明 |
|---|---|---:|---|---|---|
| `id` | `BIGINT` | 是 | identity | 主键 | 主键 ID，允许后端显式写入 |
| `data_id` | `INTEGER` | 是 | 无 | `CHECK data_id = 210` | 数据 ID，固定为 `0xD2` |
| `time_stamp` | `BIGINT` | 是 | 无 | `CHECK time_stamp > 0` | 数据发送时间戳，Unix 秒 |
| `air_id` | `VARCHAR(50)` | 是 | 无 | 非空字符串 | 飞机注册号/机尾号 |
| `flight_num` | `VARCHAR(50)` | 是 | `''` | 不为 `NULL` | 航班号，未知时为空字符串 |
| `received_at` | `TIMESTAMPTZ` | 是 | 当前时间 | 无 | 地面系统接收时间 |
| `utc` | `VARCHAR(30)` | 否 | `NULL` | 无 | 原始 UTC 时间字符串 |
| `latitude` | `DOUBLE PRECISION` | 否 | `NULL` | 空值或 `-90 <= latitude <= 90` | A628 飞行状态纬度，单位 degrees |
| `longitude` | `DOUBLE PRECISION` | 否 | `NULL` | 空值或 `-180 <= longitude <= 180` | A628 飞行状态经度，单位 degrees |
| `altitude` | `DOUBLE PRECISION` | 否 | `NULL` | 无 | A628 飞行状态高度，单位 ft |
| `door` | `INTEGER` | 否 | `NULL` | 空值或在 `-1, 0, 1` | 舱门离散状态 |
| `decomp` | `INTEGER` | 否 | `NULL` | 空值或在 `-1, 0, 1` | 释压离散状态 |
| `wow` | `INTEGER` | 否 | `NULL` | 空值或在 `-1, 0, 1` | 轮载状态 |
| `pa` | `INTEGER` | 否 | `NULL` | 空值或在 `-1, 0, 1` | PA 离散状态 |
| `time_to_go` | `INTEGER` | 否 | `NULL` | 空值或 `time_to_go >= 0` | 预计到达剩余时间，单位分钟 |
| `distance_to_go` | `INTEGER` | 否 | `NULL` | 空值或 `distance_to_go >= 0` | 预计到达剩余距离，单位 miles |
| `or_airport` | `VARCHAR(10)` | 否 | `NULL` | 无 | 起飞机场代码 |
| `des_airport` | `VARCHAR(10)` | 否 | `NULL` | 无 | 目的机场代码 |

### 4.3 约束

| 约束名 | 类型 | 字段 | 规则 |
|---|---|---|---|
| `pk_flight_status` | 主键 | `id` | 全表唯一 |
| `ck_flight_status_data_id` | 检查 | `data_id` | 必须等于 `210` |
| `ck_flight_status_time_stamp` | 检查 | `time_stamp` | 必须大于 `0` |
| `ck_flight_status_air_id_not_blank` | 检查 | `air_id` | 去除空白后不能为空 |
| `ck_flight_status_latitude` | 检查 | `latitude` | 为空或在 `[-90, 90]` |
| `ck_flight_status_longitude` | 检查 | `longitude` | 为空或在 `[-180, 180]` |
| `ck_flight_status_discrete_values` | 检查 | `door, decomp, wow, pa` | 每个字段为空或在 `-1, 0, 1` |
| `ck_flight_status_eta_values` | 检查 | `time_to_go, distance_to_go` | 每个字段为空或大于等于 `0` |
| `uq_flight_status_packet` | 唯一 | `data_id, air_id, flight_num, time_stamp` | 防止同一飞机同一航班同一秒重复写入飞行状态 |

### 4.4 索引

| 索引名 | 字段 | 类型 | 用途 |
|---|---|---|---|
| `idx_flight_status_flightnum_ts_desc` | `flight_num, time_stamp DESC` | B-tree | 按航班号查询最新飞行状态 |
| `idx_flight_status_airid_ts_desc` | `air_id, time_stamp DESC` | B-tree | 按飞机查询状态、排查数据 |
| `idx_flight_status_ts_airid_desc` | `time_stamp DESC, air_id` | B-tree | 按时间窗口检索状态数据 |

## 5. 表关系

| 来源表 | 目标表 | 关系类型 | 关联字段 | 是否使用外键 | 说明 |
|---|---|---|---|---|---|
| `system_position` | `flight_status` | 逻辑多对一 | 优先按 `flight_num`，必要时结合 `air_id` | 否 | 轨迹点写入频繁且可能乱序到达，不建议强制外键 |
| `flight_status` | `system_position` | 逻辑一对多 | 同一 `flight_num` 或同一 `air_id` 的时间邻近数据 | 否 | 一条状态可服务多个位置点展示 |

后端查询规则：

- 活跃快照：从 `system_position` 获取每架飞机最新位置点，再用 `flight_status.flight_num` 查询对应最新状态。
- 最新航段：按 `system_position.air_id` 查询过去 8 小时轨迹点，按 `time_stamp` 升序排序，再用速度阈值切出最新活跃航段。
- 机场名、航空公司名不在本最小 schema 中建表，由后端映射层处理。若其他项目希望后台配置化，可另建字典表，但不属于本实时轨迹最小 schema。

## 7. 后端建表检查清单

- 两张表均存在主键 `id`。
- `system_position.data_id` 固定为 `217`，`flight_status.data_id` 固定为 `210`。
- 两张表均包含 `time_stamp`、`air_id`、`flight_num`、`received_at`。
- `system_position` 必须包含经纬度、速度、航向、高度和姿态角字段。
- `flight_status` 必须包含起降机场、剩余时间和剩余距离字段。
- 已创建 `system_position(air_id, time_stamp DESC)` 和 `flight_status(flight_num, time_stamp DESC)` 索引。
- 未对两表设置硬外键，避免高频写入和乱序数据被外键阻塞。
- 样例数据写入后，按 `flight_num = 'MU6666'` 可从 `flight_status` 找到航班补充信息，按 `air_id = 'B-5688'` 可从 `system_position` 找到两条轨迹点。
