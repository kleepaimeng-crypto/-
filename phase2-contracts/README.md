# Phase 2 共用契约

本目录是“流量统计”和“智慧舷窗”并行开发的唯一共用契约，不按开发人员拆分文档，也不覆盖根目录第一版文档。

## 阅读顺序

1. `SPEC.md`：范围、行为和验收标准。
2. `API.md`：REST 接口、参数和响应字段。
3. `schema.md`：现有表的使用及查询口径。
4. `SLICES.md`：依赖、并行边界和完成标准。
5. `AGENTS.md`：本目录及 Phase 2 实现约束。

需求来源以 `docs/前中后舱网联数据分析平台.docx` 为准；`docs/前中后舱网联数据平台 6.png` 只提供深色视觉基线。座位布局使用模拟器 320 个终端、32 排、每排 10 座的规则，不采用界面稿的座位数量，也不使用图片素材。

## 契约优先级

这两个功能发生冲突时依次采用：Phase 2 `SPEC.md`、`API.md`、`schema.md`、`SLICES.md`。未在本目录授权的事项仍遵守根级契约。发现字段或行为无法同时满足时停止实现并同步修订本目录全部受影响文档。
