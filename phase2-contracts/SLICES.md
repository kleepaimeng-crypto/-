# Phase 2 垂直切片

## P2-00：契约与共享框架

- 建立本目录完整契约，在根 `AGENTS.md` 增加最小授权入口。
- 提取共享应用外壳和主导航，新增三条产品路由；根路径重定向数据管理。
- 建立可暂停的 5 秒轮询：立即加载、单请求在途、恢复后立即刷新、卸载取消。
- 完成标准：现有数据管理行为不回归，两条新路由可独立挂载，公共类型和轮询测试通过。

## P2-01A：流量统计

- 独立实现 options、overview、terminals 和 terminal detail 的 DAO、Service、Controller 与 DTO。
- 实现筛选、指标、应用排行、服务端分页终端列表和详情抽屉，并覆盖四态。
- 刷新保留筛选、分页、排序、选中终端及详情。
- 完成标准：真实模拟器数据展示高清视频、音乐、网页浏览，聚合口径与 `schema.md` 一致，后端和前端测试通过。

## P2-01B：智慧舷窗

- 与 P2-01A 并行，独立实现最新完整快照查询、派生侧别、区域汇总和 DTO。
- 实现 32×10 座位、左右各 100 舷窗及唯一整体滚动容器，覆盖四态和详情。
- 刷新保持 `windowId` 选择及滚动位置；来源 ID 跳转现有只读详情。
- 完成标准：200 个 ID 唯一完整、分区和侧别正确，布局及刷新行为测试通过。

## P2-02：联调验收

- 合并两条并行线，解决共享导航和类型冲突，不扩大功能范围。
- 使用真实模拟器报文执行 API、页面、轮询、错误和空数据验收。
- 执行 Docker Compose 配置检查、后端 `test/verify`、前端 `typecheck/test/build`，在 1920×1080 和 1366 宽度检查布局。
- 登记但不顺手处理旧安全、登录审计和通用测试基线问题。

## 并行边界

- P2-00 必须先形成可合并基线；P2-01A 与 P2-01B 此后并行。
- 两条功能线不得修改对方 Controller、Service、DTO、视图或测试；共享代码变更先合入公共基线。
- P2-02 之前不得把任一功能标记完成；没有验证结果不算完成。
