# 乘客实时动态切片规则

- 本目录规则只授权“乘客实时动态”单页面；根级登录、数据管理、安全和验证规则继续有效。
- 页面接口统一使用 `/api/v1/passenger-realtime/**`，禁止新增 `/traffic-statistics`、统计报告或通用分析接口。
- 影音排行必须来源于 `ife_633_behavior` 和 `ife_cockrell_behavior`，禁止使用 `traffic_record`、随机数或固定业务假数据。
- 排行值是类型出现次数的整数，不是 Mbps、字节数、播放时长或历史累计带宽。
- 座位采用模拟器 320 个终端、32 排、每排 `A B C D E F G H J K` 十座。
- 左右舷窗各 100 个，与中心座位区放在同一个纵向滚动容器中；舷窗与座位不要求一一对应。
- 自动刷新默认 5 秒，可暂停；不得关闭乘客/舷窗详情、清除选择或重置滚动位置。
- 原始报文只允许跳转现有数据详情只读查看，不在本页面编辑。
- 不修改模拟器报文，不新增业务表；Schema 变化必须先修订本目录契约并获得确认。
- 不实现驾驶舱视频、乘客推荐、舷窗控制、数据统计或用户管理。
