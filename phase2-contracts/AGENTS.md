# Phase 2 开发约束

本文件约束流量统计和智慧舷窗的前后端实现。所有开发者使用本目录同一套契约。

- 仅新增只读查询能力；不修改模拟器报文结构、UDP 入库、登录、数据管理、导入导出和保留策略。
- 不新增表、字段、索引或 Flyway 迁移。若现有查询达不到性能要求，先提供执行计划证据，再变更契约。
- 丢包率延期：模拟器无真实丢包字段，禁止以 `record_status`、包数或随机值推算。
- 页面数据必须来自 API，不写固定业务假数据，不使用图片素材。
- 两个功能使用独立后端包、前端视图和测试；共享导航、认证请求封装和轮询能力，避免并行修改同一页面。
- 自动刷新默认 5 秒，可暂停；上一请求未结束时不得重复发起。刷新不得关闭详情、清除选择或重置滚动位置。
- 智慧舷窗的左侧舷窗、32×10 座位区、右侧舷窗必须处于同一个纵向滚动容器；只允许一个纵向滚动源。
- 原始报文只通过现有数据详情只读查看，Phase 2 不提供编辑入口。

每个切片完成后运行后端测试与验证、前端类型检查/测试/构建，并检查无关 diff 和契约偏离。
