# 乘客实时动态产品规格

## 1. 页面组成

页面由四个区域构成：

1. 乘客影音统计：视频类型排行和音乐类型排行。
2. 驾驶舱实时监控：保留界面面板，显示“接口暂未接入”，不轮询、不播放视频。
3. 中部舱位图：飞机、237 个座位及左右共 116 个舷窗，可整体上下滚动。
4. 最右侧乘客观看详情：每页 4 人，可分页查看全部 237 名乘客。

## 2. 影音类型排行

- 合并 633 IFE 和科克瑞尔 IFE，对每名乘客分别保留最新视频行为和最新音乐行为。
- 视频取 `contentType`，音乐取 `musicType`；按 `/` 拆分、trim、去空并在单条行为内去重。
- 每名乘客的最新媒体状态包含某类型时，该类型计 1 次；按次数降序、类型名称升序排列。
- 显示整数次数和相对条形长度，不显示带宽、流量、播放时长或 Mbps。
- 数据和统计口径必须与 `simulator/receiver_server.py` 的视频/音乐排行一致。

## 3. 舱位图与舷窗

- 使用 `CA_332-1 2.svg` 作为飞机底图，使用 `plane-seat-overlay-renamed.svg` 的 237 个座位分组作为交互层。
- 业务座位号保持 `11A`；前端内部转换为 `A11` 定位对应 SVG 座位，不向接口增加映射字段。
- 舱位图、左侧 58 个舷窗和右侧 58 个舷窗处于同一个纵向滚动容器。
- 每个舷窗显示编号、亮度 0–10、连接状态及 `NORMAL/FAULT/TEST`；亮度即界面“透光度”。
- ID 1–58 为左侧，59–116 为右侧；每侧序号 1–9 前舱、10–32 中舱、33–58 后舱。
- 自动刷新不得改变滚动位置或当前选中座位。

## 4. 乘客观看详情与双向定位

- 右侧按固定座位顺序分页，每页恰好 4 人，最后一页可少于 4 人。
- 每项展示：座位号、行为类别、观看/浏览标题、视频或音乐类型、播放动作、浏览域名/URL、当前带宽 Mbps、当前窗口字节数和更新时间；无值显示“暂无”，不得猜测。
- 点击中部座位：选中座位，查询其所在详情页，右侧跳到该页并高亮对应乘客。
- 点击右侧详情：选中对应座位，中部滚动容器将该 SVG 座位定位到可视区域中央并高亮。
- 手动翻页只改变右侧列表；只有点击详情项才触发中部定位。
- 选中状态以 `seatNo` 为唯一键；5 秒刷新期间保持详情页、选中项和滚动位置。

## 5. 数据状态

- initial loading：首次快照和首个详情页加载中。
- error：首次请求失败时显示重试；后台刷新失败时保留旧数据并标记可能过期。
- empty：尚无 IFE/舷窗数据时分别显示空态，舱位 SVG 仍可查看。
- normal：显示真实排行、座位状态、舷窗和分页详情。

## 6. 验收标准

- 页面显示视频/音乐整数次数排行，不出现带宽排行。
- 中部能滚动查看 237 个编号座位和 116 个舷窗。
- 点击任意有乘客的座位可定位右侧正确页和条目；点击右侧任意条目可反向定位座位。
- 右侧每页 4 人，总计 237 人时为 60 页，最后一页 1 人。
- 浏览行为显示域名/URL和累计字节；当前带宽只来自最新流量窗口。
- 驾驶舱监控面板存在但不发起接口请求。
