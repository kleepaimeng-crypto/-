# 乘客实时动态产品规格

## 1. 页面组成

页面由四个区域构成：

1. 乘客影音统计：视频类型排行和音乐类型排行。
2. 驾驶舱实时监控：保留静态面板，不请求接口、不播放视频。
3. 中部舱位图：飞机、237 个座位及左右共 116 个舷窗，可整体上下滚动。
4. 最右侧乘客观看详情：连续滚动查看全部 237 名乘客的最新状态。

## 2. 影音类型排行

- 临时合并 633 IFE 和科克瑞尔 IFE，对每名乘客分别保留最新视频行为和最新音乐行为。
- 视频取 `contentType`，音乐取 `musicType`；按 `/` 拆分、trim、去空并在单条行为内去重。
- 每名乘客的最新媒体状态包含某类型时，该类型计 1 次；按次数降序、类型名称升序排列。
- 标题显示类型累计次数，排行显示整数次数和相对条形长度，不显示带宽、流量或 Mbps。
- 数据口径与 `simulator/receiver_server.py` 的视频/音乐排行一致。

## 3. 舱位图与舷窗

- 使用现有飞机底图、`plane-seat-overlay-renamed.svg` 的 237 个座位分组和 `airplane_windows.svg` 的 116 个舷窗分组。
- 模拟器、数据库、接口和 SVG 统一使用“字母+排号”的座位号，例如 `A11`；不增加映射字段。
- 舱位图、左侧 58 个舷窗和右侧 58 个舷窗处于同一个纵向滚动容器。
- 每个舷窗按亮度 0–10 改变 SVG 明暗并显示透光度；断连、`FAULT`、`TEST` 分别使用灰、红、橙状态色。
- ID 1–58 为左侧，59–116 为右侧；每侧序号 1–17 前舱、18–37 中舱、38–58 后舱。
- 自动刷新不得改变滚动位置或当前选中座位。

## 4. 乘客观看详情与双向定位

- 后端一次返回按固定座位顺序排列的 237 条精简最新状态，不提供分页和座位定位接口。
- 每项展示座位号、行为类别、观看/浏览标题、视频或音乐类型、播放动作、浏览域名/URL、当前带宽、当前窗口字节数和更新时间；无值显示“暂无”。
- 点击中部座位：右侧连续列表滚动到对应乘客并高亮。
- 点击右侧详情：中部滚动容器将对应 SVG 座位定位到可视区域中央并高亮。
- 选中状态以 `seatNo` 为唯一键；5 秒刷新期间保持选中项和两侧滚动位置。

## 5. 数据状态与验收

- 页面各数据区域完整处理 initial loading、error、empty 和 normal；后台刷新失败时保留上一次成功结果。
- 页面显示视频/音乐整数次数排行，不出现带宽排行。
- 中部能滚动查看 237 个编号座位和 116 个舷窗，透光度与 SVG 编号逐一对应。
- 点击任意座位与右侧任意详情均可双向定位，不发起额外定位请求。
- 浏览行为显示累计字节；当前带宽只来自最新流量窗口。
- 驾驶舱监控面板存在但不发起接口请求。
