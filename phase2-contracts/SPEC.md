# 乘客实时动态产品规格

## 1. 目标与范围

在主导航开放“乘客实时动态”，提供同一页面内的乘客影音类型排行、全机座位/乘客状态、智慧舷窗状态和选中对象详情。对应 Word 需求 017、018、019、021、022。

本期明确不开发“数据统计”页面及其流量、吞吐、带宽、丢包接口；不实现个性化推荐、驾驶舱视频播放、舷窗写控制或固定图片内容。

## 2. 乘客影音统计

- 左上角展示“乘客影音统计”，分为视频类型排行和音乐类型排行。
- 视频来源为两类 IFE 数据中的 `MOVIE_PLAY.behavior_detail.contentType`；音乐来源为 `MUSIC_PLAY.behavior_detail.musicType`。
- 模拟器口径：合并 633 IFE 与科克瑞尔 IFE 后，对每名乘客分别取最新一条视频行为和最新一条音乐行为；按 `/` 拆分类型、去除空白，并对同一乘客同一媒体状态中的重复类型去重。
- 排名值为类型出现次数的整数；每个乘客的最新媒体状态包含某类型时，该类型增加 1 次。禁止显示 Mbps、字节、流量或播放时长。
- 视频总次数为具有有效最新视频类型状态的乘客数；音乐总次数同理。排行按次数降序、类型名称升序稳定排序。
- 正常模拟器数据至少覆盖视频的奇幻、爱情、武侠、科幻等类型，以及音乐的民谣、乡村、舞曲、古典、摇滚等类型；类型必须由真实报文动态产生。

## 3. 客舱与乘客详情

- 中部展示 32 排、每排 10 座，座位字母固定为 `A B C D E F G H J K`。
- 座位根据该乘客最近一条 IFE 行为显示视频、音乐或其他行为状态；没有行为数据时显示空闲/未知，不制造假行为。
- 点击座位后在右侧保持显示乘客详情：座位、乘客/设备标识、媒体类别、标题、类型、动作、事件时间和来源记录 ID。
- 来源记录 ID 可跳转 `/data-management?recordId={sourceRecordId}` 打开现有只读数据详情。

## 4. 智慧舷窗

- 展示 200 个舷窗的编号、前/中/后舱分区、亮度 0–10、连接状态及 `NORMAL/FAULT/TEST` 状态。
- 每四个连续 ID 为一组，前两个归左侧、后两个归右侧，共左右各 100 个；舷窗沿座位区高度均匀分布，不要求和座位一一对应。
- 汇总展示平均亮度、断连数、故障数和测试状态数。
- 点击舷窗显示最近更新时间和来源记录 ID，并可跳转只读数据详情。
- 左舷窗、中心座位和右舷窗必须位于一个整体纵向滚动容器，只允许一个纵向滚动源。

## 5. 状态与刷新

- 页面完整处理 initial loading、error、empty 和 normal；已有成功数据后的刷新失败保留旧数据并显示“数据可能已过期”。
- 默认每 5 秒刷新，可暂停和手动刷新；上一请求未完成时不重复发起。
- 刷新不得修改当前选中座位、选中舷窗、详情面板状态或整体滚动位置；对象消失时才关闭对应详情并给出提示。

## 6. 验收标准

- 主导航仅新增“乘客实时动态”可用入口，不开放“数据统计”。
- 影音排行全部为整数次数，页面和接口均不出现 Mbps、吞吐、流量或丢包字段。
- 使用模拟器数据可看到视频、音乐排行、32×10 座位和完整 200 个舷窗；左右各 100 个。
- 三列共用一个滚动条，暂停/恢复刷新有效，刷新期间选择和详情保持。
