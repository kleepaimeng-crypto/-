# 数据记录编辑弹窗：原始 JSON 只读查看

## 目标

在数据管理页的“编辑管理信息”弹窗中，让管理员可就地查看该记录的完整原始报文 JSON，同时继续只编辑现有元数据字段。

## 范围

- 仅修改 `frontend/src/views/WorkspaceView.vue`、对应样式和前端测试。
- 复用既有 `GET /api/v1/data-records/{recordId}` 详情接口。
- 不修改数据库、Flyway、后端接口或保存请求。
- 原始报文、解析摘要仍然不可编辑且不会被提交。

## 交互设计

1. 用户点击一条记录的“编辑”。
2. 现有元数据表单立即打开；同时按需读取该记录详情。
3. 在表单下方显示折叠区“原始报文 JSON（只读）”。
4. 默认折叠；展开后以格式化 JSON 展示合法 `rawPayload`，解析失败记录则展示 `rawText`。
5. JSON 区域有固定最大高度并在内部滚动，避免大批量 IFE 报文撑高弹窗。
6. 详情读取失败时，在折叠区显示错误提示，不影响元数据编辑和保存。
7. 保存仍只发送现有 `MetadataUpdatePayload`，关闭弹窗时清理本次详情状态。

## 错误与边界

- 报文可能较大，前端不在列表加载时预取，只在打开编辑弹窗时请求。
- 关闭后才返回的旧响应不得写入下一次编辑的记录状态。
- 空 JSON / 文本使用明确的只读空态。
- 不展示密码、JWT 等敏感数据；当前 UDP 原始报文契约中不包含这些字段。

## 验收

- 编辑任一合法 JSON 记录时，可展开查看格式化 JSON。
- 编辑解析失败记录时，可查看原始文本。
- IFE 大报文在独立滚动区显示，元数据表单和保存按钮仍可操作。
- 详情读取失败不阻止保存元数据。
- 保存请求不包含 `rawPayload`、`rawText` 或 `parsedSummary`。
