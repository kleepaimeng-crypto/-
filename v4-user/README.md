# 用户管理阶段契约

本目录定义“前中后舱网联数据显示平台”用户管理页面的产品、接口、数据和实施边界。当前状态为：用户管理垂直切片已进入实现与验收阶段。

## 必读顺序

1. `SPEC.md`：产品范围、角色状态、交互和验收标准。
2. `API.md`：用户管理 REST API。
3. `schema.md`：`app_user` 演进方案和 Flyway 迁移要求。
4. `SLICES.md`：实施顺序、文件范围和每个切片的完成标准。
5. `AGENTS.md`：开发约束和禁止事项。

同时必须阅读当前项目基础契约：

- `../v1-docs/SPEC.md`
- `../v1-docs/API.md`
- `../v1-docs/schema.md`
- `../v1-docs/AGENTS.md`

## 授权边界

本目录只覆盖基础契约中“暂不开发用户管理”的限制，授权实现：

- 用户分页列表。
- 用户添加、编辑和软删除。
- `SUPER_ADMIN`、`ADMIN`、`USER` 三种角色。
- `ACTIVE`、`PENDING`、`FROZEN`、`DELETED` 四种状态。
- 用户管理专属访问控制和审计。
- 参考图 `docs/前中后舱网联数据平台 5.png` 对应的用户管理页面。

本期不实现完整 RBAC、权限点配置、角色配置页面、组织架构、批量用户操作、邮件激活、密码找回、单独的重置密码流程或第三方身份源。

## 核心决策

- 复用现有 `app_user`，不新增角色表和权限表。
- 只有 `SUPER_ADMIN` 可以访问用户管理页面和 `/api/v1/users/**`。
- 删除为不可恢复的软删除，不物理删除用户。
- 现有 `ADMIN` 数据迁移为 `SUPER_ADMIN`，现有 `DISABLED` 数据迁移为 `FROZEN`。
- 数据管理记录写操作区分 `ADMIN` 与 `USER`：`SUPER_ADMIN`、`ADMIN` 可以编辑和删除数据记录，`USER` 只能查看，前端写按钮置灰，后端写接口返回 403。
- 参考图中的复选框没有对应批量动作，本期不实现无效复选框或批量接口。
